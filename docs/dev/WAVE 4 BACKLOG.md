# Wave 4 backlog (tracking)

**Purpose:** Sprint and PR tracking only. Do **not** duplicate full objectives, scope, acceptance criteria, tests, or Cursor prompts here.

**Maintenance:** After each merged story (or agreed milestone), update the **Status** and **Notes** columns for that row. Use `Done` only when the ticket’s acceptance criteria in the pack are met.

**Canonical source:** Every Wave 4 story’s full ticket lives in [`Wave 4 Ticket Pack.md`](../prompts/Wave%204%20Ticket%20Pack.md) under the matching `**PROC-STORY-xxx**` heading.

**Execution order:** Follow the pack’s **Recommended Wave 4 Build Order** and **Recommended Cursor Execution Order**: evaluation **057–072**, award **073–085**, contract **086–099**.

- **Upstream:** Wave 3 procurement opening chain complete for the target environment ([`WAVE 3 BACKLOG.md`](WAVE%203%20BACKLOG.md) **PROC-STORY-048–056**; tender/bid models **023–047** as needed for links).
- **App:** `kentender_procurement` — same layout as earlier waves ([`kentender_procurement/`](../../kentender_procurement/)); run `bench migrate` as DocTypes and script reports land.
- **Cross-app:** **PROC-STORY-097** (contract variation) expects **budget validation** integration with `kentender_budget` per the pack; keep dependencies acyclic ([`app-dependencies-and-boundaries.md`](../architecture/app-dependencies-and-boundaries.md)).
- **Tests:** `bench run-tests --app kentender_procurement` (and any other app a story touches); keep green per story.
- **Derived / service-owned DocTypes:** For **PROC-STORY-064** (and similar), define write policy from the first merge (e.g. server-only updates, permissions, or guards) so desks cannot populate aggregation rows before **PROC-STORY-070** — see pack constraints.
- **Minimal Golden UAT:** Extend canonical JSON / loaders / `verify_minimal_golden` when a story needs a deterministic user-visible row; otherwise note skip in **Notes**. Follow [`uat/seed_packs/minimal_golden/README.md`](../../uat/seed_packs/minimal_golden/README.md) and existing minimal-golden rules in the repo.
- **Parallel UI / UAT:** Workspaces, seed packs, and acceptance journeys remain in [`UAT AND WORKSPACE IMPLEMENTATION BACKLOG.md`](../testing/UAT%20AND%20WORKSPACE%20IMPLEMENTATION%20BACKLOG.md) (separate from this table). Pack end-state UAT hints: **AT-EVAL-001**, **AT-AWD-001**, **AT-CON-001**.
