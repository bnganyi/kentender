# Wave 5 backlog (tracking)

**Purpose:** Sprint and PR tracking only. Do **not** duplicate full objectives, scope, acceptance criteria, tests, or Cursor prompts here.

**Maintenance:** After each merged story (or agreed milestone), update the **Status** and **Notes** columns for that row. Use `Done` only when the ticket’s acceptance criteria in the pack are met.

**Canonical source:** Every Wave 5 story’s full ticket lives in [`Wave 5 Ticket Pack.md`](../prompts/Wave%205%20Ticket%20Pack.md) under the matching `**PROC-STORY-xxx**` or `**GOV-STORY-xxx**` heading.

**Execution order:** Follow the pack’s **Recommended Wave 5 Build Order** (same sequence as **Step #** below): **Inspection & Acceptance** **100–116**, **Deliberations** **GOV-001–011**, **Complaints & Disputes** **GOV-012–025**.

**Prerequisites**

- **Upstream:** Wave 4 complete for the target environment ([`WAVE 4 BACKLOG.md`](WAVE%204%20BACKLOG.md) **PROC-STORY-057–099**). Inspection stories link to **Procurement Contract** and related contract scope DocTypes (**PROC-STORY-086–089**); keep naming aligned with the pack (avoid clashing with core Frappe / ERPNext DocType names).
- **Apps:** **`kentender_procurement`** — **EPIC-PROC-009** (inspection & acceptance), same layout as earlier procurement waves ([`kentender_procurement/`](../../kentender_procurement/)). **`kentender_governance`** — **EPIC-GOV-001** and **EPIC-GOV-002** (deliberations; complaints). Governance is a scaffold today; extend it per [`application-package-layout.md`](../architecture/application-package-layout.md). Run **`bench migrate`** as DocTypes and script reports land.
- **Cross-app:** Respect the KenTender DAG ([`app-dependencies-and-boundaries.md`](../architecture/app-dependencies-and-boundaries.md)): **`kentender_governance`** may depend on **`kentender`** only among KenTender apps — no direct imports of procurement **`services/`** internals. Integrate with contracts / procurement via **`api/`**, documented **`frappe.call`** entry points, or shared facades in **`kentender`** when a story explicitly requires it.
- **Tests:** `bench run-tests --app kentender_procurement` and `bench run-tests --app kentender_governance` (and any other app a story touches); keep green per story.
- **Minimal Golden UAT:** Extend canonical JSON / loaders / `verify_minimal_golden` when a story needs a deterministic user-visible row; otherwise note skip in **Notes**. Follow [`uat/seed_packs/minimal_golden/README.md`](../../uat/seed_packs/minimal_golden/README.md).
- **Parallel UI / UAT:** Workspaces and acceptance journeys remain in [`UAT AND WORKSPACE IMPLEMENTATION BACKLOG.md`](../testing/UAT%20AND%20WORKSPACE%20IMPLEMENTATION%20BACKLOG.md) (separate from this table).
