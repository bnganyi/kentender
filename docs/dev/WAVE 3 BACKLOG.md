# Wave 3 backlog (tracking)

**Purpose:** Sprint and PR tracking only. Do **not** duplicate full objectives, scope, acceptance criteria, tests, or Cursor prompts here.

**Maintenance:** After each merged story (or agreed milestone), update the **Status** and **Notes** columns for that row. Use `Done` only when the ticket’s acceptance criteria in the pack are met.

**Canonical source:** Every Wave 3 story’s full ticket lives in [`Wave 3 Ticket Pack.md`](../prompts/Wave%203%20Ticket%20Pack.md) under the matching `**PROC-STORY-xxx**` heading.

**Execution order:** Follow the pack’s **Recommended Wave 3 Build Order** (tender **023–035**, bid submission **036–047**, bid opening **048–056**).

**Prerequisites**

- **Upstream:** Wave 2 procurement planning chain complete for the target environment ([`WAVE 2 BACKLOG.md`](Wave%202%20BACKLOG.md) steps **011–022**).
- **App:** `kentender_procurement` — same layout as earlier waves ([`kentender_procurement/`](../../kentender_procurement/)); run `bench migrate` as DocTypes land.
- **Tests:** `bench run-tests --app kentender_procurement`; keep green per story where applicable.
- **Minimal Golden UAT:** Per [`Wave 3 Ticket Pack.md`](../prompts/Wave%203%20Ticket%20Pack.md) section **Wave 3 — Repo hygiene and Minimal Golden UAT** — extend canonical JSON/loaders/verify when the story affects golden data; on follow-along sites run `seed_minimal_golden_console` and `verify_minimal_golden_console` (see [`uat/seed_packs/minimal_golden/README.md`](../../uat/seed_packs/minimal_golden/README.md)).
- **Parallel UI / UAT:** [`UAT AND WORKSPACE IMPLEMENTATION BACKLOG.md`](../testing/UAT%20AND%20WORKSPACE%20IMPLEMENTATION%20BACKLOG.md) (separate from this table).
